# My First Next.js App

## Running the Development Server

After creating a Next.js project, you can start the development server using:

```bash
npm run dev
```

This will start the application locally and you can view it in the browser (usually at `http://localhost:3000`).

---

# Root Page

In a Next.js project using the **App Router**, the root page is located at:

```tsx
app/page.tsx
```

This file represents the **home route (`/`)** of the application.
All code related to the **root page UI** is written inside this file.

---

# Project File & Folder Structure

After creating a Next.js project, several files and folders are automatically generated.

## Important Files

### 1. `package.json`

Contains:

- Project metadata
- Installed dependencies
- Scripts used to run the project

Example scripts:

```json
"scripts": {
  "dev": "next dev",
  "build": "next build",
  "start": "next start"
}
```

---

### 2. `tsconfig.json`

Configuration file for **TypeScript**.

It defines:

- TypeScript compiler options
- Path aliases
- Type checking behavior

---

### 3. `eslint.config.mjs`

Configuration file for **ESLint**.

ESLint helps maintain **clean and consistent code** by identifying errors and enforcing coding standards.

---

### 4. `tailwind.config.ts`

Configuration file for **Tailwind CSS**.

It allows you to customize:

- Themes
- Colors
- Plugins
- Content paths

---

### 5. `postcss.config.ts`

Used by **PostCSS** to process Tailwind CSS.

It converts Tailwind utilities into standard CSS that browsers can understand.

---

### 6. `package-lock.json`

Automatically generated by npm.

It locks the **exact versions of installed dependencies** to ensure consistent installs across environments.

---

### 7. `README.md`

Contains **documentation about the project**.

Usually includes:

- Project description
- Setup instructions
- Features
- Usage guide

---

### 8. `.gitignore`

Specifies files and folders that **Git should ignore** when committing.

Common ignored files include:

- `node_modules`
- `.next`
- environment files

---

### 9. `next-env.d.ts`

A **TypeScript declaration file** used by Next.js to recognize its internal types.

---

# Important Folders

### 1. `.next`

This folder is generated when the project is **built or run**.

It contains the compiled application used to run the project.

⚠️ This folder is automatically generated and **should not be edited manually**.

---

### 2. `node_modules`

Contains all **installed project dependencies**.

This folder is created automatically when running:

```bash
npm install
```

---

### 3. `public`

Used to store **static assets** such as:

- Images
- SVG files
- Fonts

Example:

``` tsx
public/logo.svg
public/image.png
```

These files can be accessed directly in the browser.

---

### 4. `src`

The `src` folder contains the main application code.

Inside it, you will find the **App Router directory**:

```tsx
src/app
```

Initially, it contains the following files:

#### a. `favicon.ico`

Website icon shown in the browser tab.

#### b. `globals.css`

Global CSS styles applied across the application.

#### c. `layout.tsx`

Defines the **shared layout structure** for pages.

Example:

- Header
- Footer
- Navigation

---

#### d. `page.tsx`

Represents the **main page (`/`)** of the application.

# React Server Components (RSC)

**React Server Components (RSC)** are a new architecture introduced by the React team and adopted by Next.js.

This architecture divides React components into **two types**:

1. Server Components
2. Client Components

---

## Server Components

Server components run on the **server**.

Characteristics:

- In Next.js, components are **server components by default**
- Can perform **server-side tasks**
- Can directly access databases
- Can read files from the server
- Can fetch data securely

Example tasks:

- Fetching data from a database
- Reading files from the filesystem

---

## Client Components

Client components run in the **browser**.

To create a client component, add this at the top of the file:

```tsx

"use client";
```

Characteristics:

- Can use **React hooks** such as:
  - `useState`
  - `useEffect`

- Handles **interactivity and user events**
- Cannot directly access the database

Client components are similar to **traditional React components**.
